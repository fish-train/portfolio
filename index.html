Обо мне

Меня зовут Александр, я технический писатель.

Разрабатываю документацию на программное обеспечение 9 лет. Помогаю софтверным компаниям создавать удобную и понятную документацию для пользователей, администраторов и разработчиков.

Я не только работаю с текстом, но и:

  • внедряю технологии и выстраиваю процессы документирования,
  • обучаю сотрудников,
  • управляю командой,
  • пишу тексты для интерфейсов,
  • автоматизирую разработку документации,
  • помогаю писать инженерам о технологиях и продуктах компании,
  • развиваю базу знаний.

Мне интересны проекты, в которых используется или внедряется принцип docs as code.

Учебные проекты
Docs as Code

Цель: познакомиться с принципом Docs as Code, изучить технологии и инструменты:

  • Foliant – инструмент создания документации,
  • Markdown – язык разметки,
  • Git – система контроля версий,
  • GitHub Pages – хостинг,
  • Travis CI – веб-сервис для сборки и тестирования ПО,
  • VS Code – текстовый редактор.

Дополнительно рассмотрел работу с Docker, Swagger и GostDOWN.

Sphinx и reStructuredText

Цель: познакомиться с Sphinx и reStructuredText.

Задачи:

  • Создать проект Sphinx для документации на языке разметки reStructuredText.
  • Генерировать документацию в формате HTML.
  • Генерировать документацию в формате PDF.
  • Хранить исходники в Git.
  • Публиковать сайт с документацией на ReadTheDocs
